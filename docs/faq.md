# FAQ

## Harness Docs는 일반 CMS와 무엇이 다른가요?

Harness Docs는 단순히 문서를 게시하는 도구보다, 역할별 제품 문서를 운영하는 흐름에 더 초점을 둡니다. 요구사항, 사용자 흐름, 기술 설계, 정책 문서를 한 워크스페이스 안에서 연결해 다루려는 점이 다릅니다.

## GitHub를 꼭 써야 하나요?

현재 방향에서는 GitHub를 문서 배포 채널로 사용하는 것을 기본 전제로 둡니다. 다만 GitHub가 문서 작업 자체의 기준점은 아니고, 최종 배포 채널 역할에 가깝습니다.

## 내부 개발 문서에도 쓸 수 있나요?

쓸 수 있습니다. 다만 개인 메모보다는 팀이 함께 관리하는 제품 문서에 더 맞춰져 있습니다. 특히 요구사항, 사용자 흐름, 기술 설계, 정책 문서처럼 서로 연결된 문서에 적합합니다.

## 어떤 문서를 넣는 것이 좋나요?

아래 문서가 특히 잘 맞습니다.

- 제품 요구사항 문서
- 사용자 흐름 문서
- 기술 설계 문서
- 정책과 의사결정 문서

## 누가 이 문서를 관리하나요?

PM, 디자이너, 개발자, 리드처럼 제품 판단에 직접 참여하는 역할이 함께 관리하는 형태가 잘 맞습니다.

## AI는 어떤 역할을 하나요?

AI는 초안을 만들고 문서 구조를 제안하는 보조 역할에 가깝습니다. 최종 메시지와 공개 판단은 사람이 합니다.
