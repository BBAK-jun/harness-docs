# FAQ

## Harness Docs는 일반 CMS와 무엇이 다른가요?

Harness Docs는 단순히 문서를 게시하는 도구보다, 고객-facing 문서를 운영하는 흐름에 더 초점을 둡니다. 문제 설명, 사용 가이드, 검토, 배포 준비 상태를 한 워크스페이스 안에서 연결해 다루려는 점이 다릅니다.

## GitHub를 꼭 써야 하나요?

현재 방향에서는 GitHub를 문서 배포 채널로 사용하는 것을 기본 전제로 둡니다. 다만 GitHub가 문서 작업 자체의 source of truth는 아니고, 최종 배포 채널 역할에 가깝습니다.

## 내부 개발 문서에도 쓸 수 있나요?

쓸 수는 있지만, 제품 방향은 내부 개발 문서보다 고객-facing 문서 운영에 더 맞춰져 있습니다. 즉 "왜 필요한가"와 "어떻게 쓰는가"를 함께 전달하는 문서에 더 적합합니다.

## 어떤 문서를 넣는 것이 좋나요?

아래 문서가 특히 잘 맞습니다.

- 고객 문제 설명 문서
- getting started
- how-to guide
- 운영 정책
- FAQ

## 누가 이 문서를 관리하나요?

PM, 솔루션/CS 팀, 운영팀, 교육/마케팅 팀처럼 고객과 제품 사이 설명 책임이 있는 역할이 함께 관리하는 형태가 잘 맞습니다.

## AI는 어떤 역할을 하나요?

AI는 초안을 만들고 문서 구조를 제안하는 보조 역할에 가깝습니다. 최종 메시지와 공개 판단은 사람이 합니다.
