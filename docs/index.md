## 왜 Harness Docs인가

Harness Docs는 단순한 Markdown 편집기나 GitHub 동기화 도구가 아닙니다. 이 제품은 PM, 디자인, 엔지니어링이 역할별 문서를 나눠 관리하면서도, 하나의 워크스페이스와 하나의 운영 체계 안에서 문서를 함께 다루기 위한 시스템입니다.

핵심 원칙:

- 어떤 문서가 최신인지, 어떤 문서가 다시 확인이 필요한지는 앱에서 더 쉽게 파악할 수 있습니다.
- 문제가 있는 문서를 숨기지 않지만, 급한 상황에서 팀이 판단하고 계속 진행할 여지는 남겨둡니다.
- AI는 초안 작성과 정리에 도움을 주지만, 최종 판단과 책임은 사람에게 남겨둡니다.
- 서로 연결된 문서가 함께 관리되므로, 한 문서의 변경이 다른 문서에 어떤 영향을 주는지 놓치기 어렵습니다.

자세한 내용은 [제품 소개](/product-overview)부터 보면 됩니다.
