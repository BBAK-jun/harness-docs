# Docs Rules

이 디렉터리는 고객에게 공개 가능한 제품 문서만 둡니다.

## Allowed Content

- 제품 소개
- 고객 문제 설명
- 사용 가이드 성격의 공개 문서
- 고객이 봐도 되는 공개 레퍼런스

## Disallowed Content

- 내부 PRD
- seed, spec source, acceptance criteria 원문
- 구현 상세, 개발 절차, 로컬 실행 방법
- API, desktop, contracts, db 내부 설계 문서

## Placement

- 공개 제품 문서는 `docs/` 아래에 둡니다.
- 내부 제품 기획 문서는 저장소 루트 또는 해당 내부 문서 위치로 둡니다.
- 개발 문서는 각 패키지의 `docs/` 아래에 둡니다.
