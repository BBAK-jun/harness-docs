# 어떻게 작동하는가

Harness Docs는 고객-facing 문서를 하나의 워크스페이스 안에서 작성, 검토, 정리하고 배포하는 흐름을 제공합니다.

## 1. 문서 워크스페이스를 만든다

팀은 먼저 하나의 문서 워크스페이스를 만듭니다.

이 워크스페이스는 고객-facing 문서를 담는 GitHub docs 저장소와 연결됩니다.

## 2. 고객이 겪는 문제를 설명하는 문서를 만든다

문서는 단순한 기능 설명에서 시작하지 않습니다.

먼저 아래를 정리합니다.

- 고객이 어떤 문제를 겪고 있었는가
- 기존 방식이 왜 불편했는가
- 우리 제품이 어떤 변화를 만들어 주는가

이 문서는 제품 가치에 공감하게 만드는 출발점 역할을 합니다.

## 3. 시작 가이드와 how-to를 연결한다

고객이 제품 가치를 이해했다면, 바로 다음 단계는 실제 사용입니다.

Harness Docs는 같은 워크스페이스 안에서 아래 문서를 함께 다루도록 돕습니다.

- getting started
- 설정 가이드
- 실제 사용 흐름 how-to
- 운영 정책
- FAQ

## 4. 문서를 검토하고 보강한다

고객-facing 문서는 문장 하나도 중요합니다.

팀은 문서를 검토하면서 아래를 확인합니다.

- 고객이 이해하기 쉬운가
- 실제 제품 흐름과 맞는가
- 빠진 단계나 혼란을 주는 표현은 없는가

## 5. GitHub로 배포한다

문서가 준비되면 GitHub는 최종 배포 채널 역할을 합니다.

즉 GitHub는 공개 docs를 전달하는 경로이고, 문서 작업 맥락과 검토 이력은 앱 안에서 관리합니다.

## 6. 계속 업데이트한다

문서는 제품 변경과 함께 계속 고쳐야 합니다.

Harness Docs는 문서를 운영 대상처럼 다루는 흐름을 지향합니다.

- 최근 바뀐 문서 확인
- 검토가 필요한 문서 확인
- 업데이트가 필요한 문서 확인
- 다시 배포할 준비 상태 확인

## 한 문장 요약

Harness Docs는 고객 문제 설명에서 시작 가이드, 검토, 배포, 업데이트까지 문서 운영 전체 흐름을 연결하는 제품입니다.
